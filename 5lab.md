## Задание 3. Удаление и восстановление пользователя

**Выполненные действия:**
1. Удалён пользователь Student1
2. Проверены права на его папку
3. Создан пользователь Student1 заново
4. Проверены права на папку для нового пользователя

**Использованные команды:**
```cmd
net user Student1 /delete
icacls "C:\NTFS_Lab\Учащиеся\Student1"
net user Student1 P@ssw0rd /add
net localgroup "Учащиеся" Student1 /add
icacls "C:\NTFS_Lab\Учащиеся\Student1"
```

**Результаты:**

**Результат проверки доступа нового пользователя:**
```cmd
runas /user:Student1 "cmd /k dir C:\NTFS_Lab\Учащиеся\Student1"
```
**Ответ:** Доступ запрещён.

**Объяснение результата:** Новый пользователь **НЕ получил** права на папку своего имени. Это происходит потому, что разрешения NTFS привязаны к **SID (Security Identifier)** — уникальному идентификатору безопасности пользователя. При удалении пользователя его SID удаляется. При создании нового пользователя с тем же именем ему присваивается **новый SID**, поэтому старые права на папку не наследуются.

## Контрольные вопросы

**1. Какое разрешение задано по умолчанию, когда том отформатирован под NTFS? Кто имеет доступ?**

**Ответ:** По умолчанию группе **Everyone** (Все) назначается разрешение **Full Control** (Полный доступ).

**2. Если пользователь имеет разрешение Write (Запись) для папки и является также членом группы с разрешением Read (Чтение) для той же папки, какими разрешениями он обладает?**

**Ответ:** Пользователь будет обладать **обоими разрешениями** (Write + Read), так как разрешения NTFS суммируются.

**3. Пользователь имеет разрешение Modify (Изменить) для папки и разрешение Read для файла. Файл копируется в эту папку. Какое разрешение для файла будет иметь пользователь?**

**Ответ:** При копировании файл наследует разрешения целевой папки. Пользователь получит разрешение **Modify** для файла.

**4. Какое из следующих разрешений NTFS для папок позволяет удалять папку?**
- a. Чтение
- b. Чтение и выполнение
- c. Изменение
- d. Администрирование

**Ответ:** **c. Изменение (Modify)** и также **Full Control** (Полный доступ).

**5. Какое разрешение NTFS для файлов следует установить для файла, если вы позволяете пользователям удалять файл, но не позволяете становиться владельцами файла?**

**Ответ:** Разрешение **Modify** (Изменение). Оно позволяет удалять и изменять файл, но не даёт прав на изменение разрешений и смену владельца (это только у Full Control).

**6. Какие объекты по умолчанию наследуют разрешения, установленные для родительской папки?**

**Ответ:** Все **файлы и подпапки**, находящиеся внутри родительской папки, наследуют её разрешения.

**7. Кто может устанавливать разрешения для отдельных пользователей и групп? (выберите все правильные ответы)**
- a. Члены группы Администраторы
- b. Члены группы Опытные пользователи
- c. Пользователи, обладающие разрешением Полный доступ
- d. Владельцы файлов и папок

**Ответ:** **a, c, d** (Администраторы, пользователи с Full Control, владельцы файлов/папок).

**8. Какую вкладку диалогового окна свойств файла или папки следует использовать для установки или изменений разрешений NTFS?**
- a. Дополнительно
- b. Разрешения
- c. Безопасность
- d. Общие

**Ответ:** **c. Безопасность (Security)**.

**9. Если вы хотите, чтобы пользователь или группа не имела доступа к определенной папке или файлу, следует ли запретить разрешения для этой папки или файла?**

**Ответ:** Можно использовать запрет (Deny), но **рекомендуется просто не предоставлять разрешение**, а не использовать Deny, так как Deny имеет приоритет над всеми Allow и может привести к неожиданным блокировкам доступа.
